阅读理解 看懂题目以后写三个限制条件就可以了。

B - Yet Another Array Partitioning Task

这题开始我就在欣赏英语了。
一开始觉得这是最大m子段和类型的问题，于是对着网上模板开始写，由于动态规划掌握得不好，始终写不出来。对着样例数据发呆的时候突然发现选出的答案和就是最大的m*k个元素和，似乎发现了什么。那么问题就变成了如何划分了。结果到比赛快结束时才有了一个从头到尾的划分方法。
夜深了，准备第二天起来再写，睡不着的时候想想越觉得这种方法是可行的。
第二天先按自己的想法写了提交，AC。

我的方案是先排序取出最大的m*k个元素累加得到最大和，然后用map建立一个映射表用于划分，然后再按原始顺序逐个扫描，如果映射值大于0，当前段选取数量加1，映射值-1，当前段选取数量达到m时，段数量+1，段选取数量置0，输出当前位置。
英文题解中提到相较于O(nlogn)的排序还有一种更优的O(n)的部分有序方法，nth_element()，它能将第n大的元素排在第n个位置，在它之前的元素均小于它，在它之后的元素均大于它。
不过根据我的实际测试排序方法对于测试时间的影响不是很大。
后面的划分方法反而影响相对更大，我写的划分方案应该相对容易理解，参考题解的划分方法可以自行理解，题解中指出事实上这种最理想的选取最大m*k个元素的方案是恒成立的。

C - Trailing Loves (or L’oeufs?)

题意大概是求出n!在b进制下的后缀0数量。
这题我没有什么好的想法，n!,b的数据范围太大了。
只能参考英文题解。
The problem can be reduced to the following: finding the maximum r that n ! is divisible by b^r.
这句话相当于将题目转化为了质因子分解问题。

以下是我的理解，十进制下末尾0数量来源与这个数能被10^r整除的最大r值，以此类比到b进制，具体的公式推导可以参考英文题解

这里只对使用的对n!求某个质因子的指数的方法做介绍
相比较于原始的累乘其复杂度显然不可行。
这里给出一种比较高效的算法:

n!=1*2*3*……*(n-2)*(n-1)*n

可以表示成所有和m倍数有关的乘积再乘以其他和m没有关系的

=(m*2m*3m*......*km)*ohter     
other是不含m因子的数的乘积   因为 km<=n 而k肯定是最大值  所以k=n/m

=m^k*(1*2*......*k)*other  

=m^k*k!*other*　

从这个表达式中可以提取出m^k，然后按照相同的方法循环下去可以求出k!中因子m的指数。每次求出m的指数的和就是n!中因子m的总指数。
