算法笔记（1）排序

前言：此系列旨在总结复习。

选择排序

以升序为例，每次找出第$i$小的元素（即$A_{i..n}$中最小元素），将其与第$i$位元素交换。
时间复杂度为$O(n^2)$，同时由于交换操作的存在，为不稳定排序。
例如：
3 4 3 2 5
2 4 3 3 5
在第一次循环中，两个3的相对顺序被破坏了。

冒泡排序

以升序为例，每次比较相邻两个元素，若前面的元素大于后面的元素，将其交换，当没有元素需要交换时，排序完成。作为一种稳定排序，它的时间复杂度为$O(n^2)$。

插入排序

将序列划分为“已排序的”和“未排序的”两部分，每次从”未排序的“元素中选择一个插入到”已排序的“元素中正确的位置。时间复杂度为$O(n^2)$，在序列几乎有序时效率很高，同时也是一种稳定排序。

计数排序

计数排序是一种非比较的时间复杂度为$O(n+w)$的稳定排序，$w$为待排序数据值域大小。

步骤如下：

  1. 计算每个数出现次数。
  2. 计算出现次数前缀和。
  3. 利用出现次数前缀和倒序计算每个数的排名。

基数排序

基数排序是将待排序的元素拆分为 $k$个关键字（比较两个元素时，先比较第一关键字，如果相同再比较第二关键字……），然后先对第 $k$关键字进行稳定排序，再对第 $k-1$关键字进行稳定排序，再对第 $k-2$关键字进行稳定排序……最后对第一关键字进行稳定排序，这样就完成了对整个待排序序列的稳定排序。

如果使用计数排序对每一关键字进行排序，基数排序的时间复杂度为$O(nk+\sum\limits_{i=1}^kw_i)$，$w_i$为第$i$关键字的值域大小。

快速排序

快速排序是一种使用分治来对序列进行排序的方法。

步骤如下：

  1. 挑选序列中的一个元素作为$pivot$。
  2. 根据与$pivot$的大小关系将序列分为前后两个子序列。
  3. 对子序列进行递归排序。

关于快速排序的实现方法有很多，这里不再赘述。一般情况下，快速排序的平均时间复杂度为$O(nlogn)$，且牺牲了稳定性。

参考快速排序的划分过程，我们也可以在$O(n)$的时间复杂度内找到一个序列中第$k$大的数。

归并排序

归并排序也是一种使用分治来对序列进行排序的方法。

步骤如下：

  1. 将序列划分为两部分。
  2. 递归地对两个子序列分别进行归并排序。
  3. 合并两个有序的子序列。

在均匀划分时，归并排序的时间复杂度为$O(nlogn)$。此外，归并排序还可以用来求出序列中逆序对的个数。

堆排序

通过数据结构$heap$来进行排序。

以大根堆为例，步骤如下：

  1. 从最后一个非叶节点开始将序列堆化，时间复杂度为$O(n)$。
  2. 对堆化序列排序，重复地取出根与最后一个节点交换，把该节点移出堆后对剩余部分重新调整堆化，时间复杂度为$O(nlogn)$。

其总体时间复杂度为$O(nlogn)$，且为不稳定排序。

桶排序

大体思想是将待排序值域分为若干个桶，将元素分配到不同的桶中，在各个桶中进行稳定的内部排序（如插入排序），最后将各个桶内数据连接得到排好序的序列。在数据均匀分布的情况下，桶排序的期望时间复杂度为$O(n)$。

希尔排序

希尔排序是插入排序的一种更为高效的改进版本，且是一种非稳定排序方法。

大体思想是从大步长到小步长（最后步长为1，即普通的插入排序）逐渐进行插入排序，其时间复杂度根据所选择不同步长序列而不同。

总结

时间复杂度 稳定性
选择排序 $O(n^2)$ ❌
冒泡排序 $O(n^2)$ ✅
插入排序 $O(n^2)$ ✅
计数排序 $O(n+w)$ ✅
基数排序 $O(nk+\sum\limits_{i=1}^kw_i)$ ✅
快速排序 $O(nlogn)$ ❌
归并排序 $O(nlogn)$ ✅
堆排序 $O(nlogn)$ ❌
桶排序 $O(n)$ ✅
希尔排序 ❌

tag上标了二分搜索，怎么理解呢，从出发点到终点的曼哈顿距离是一定的，把搜索的时间k的风向影响点求出来，根据

每走一步都受到风向的影响，可以用前缀组先得到风向的影响，分开考虑两者的运动。

看了题解后知道这是dp，但是普通的dp也接受不了这么大的复杂度。需要借助矩阵快速幂来加速计算。

参考博客

C - Meaningless Operations

一开始做的时候打表容易发现存在规律，难处理的是num=2^x-1的情况，可以预处理跑一遍然后直接查表即可。
以下为英文题解：

Denote the highest bit of a as x (that is largest
number x, such that 2^x≤a) and consider b=(2^x−1)⊕a.
It’s easy to see that if a≠2^x−1 then 0<b<a holds. In
this, gcd is maximum possible, because a&b=0 и gcd
(2^x−1,0)=2^x−1.
当a不等于2^x-1时 如a=1001 则当b=0110时取max_gcd 此时gcd(a^b,a&b)=gcd(2^x-1,0)=2^x-1

Now consider the case when a=2^x−1. This implies gcd
(a⊕b,a&b)=gcd(2^x−1−b,b)=gcd(2^x−1,b), since gcd(x,x
+y)=gcd(x,y) (for all x and y), hence it’s sufficient
to find the largest non trivial divisor of a — it
will be the desired answer.

Finding the largest divisor can be done in sqrt
time which leads to the time O(q√m) or it’s possible
to calculate answers for all a=2^x−1 beforehand.
这里推荐用预处理，我参考题解用sqrt(a)寻找最大因子时超时
了。也可能是方法不正确。

我的方案是先排序取出最大的m*k个元素累加得到最大和，然后用map建立一个映射表用于划分，然后再按原始顺序逐个扫描，如果映射值大于0，当前段选取数量加1，映射值-1，当前段选取数量达到m时，段数量+1，段选取数量置0，输出当前位置。
英文题解中提到相较于O(nlogn)的排序还有一种更优的O(n)的部分有序方法，nth_element()，它能将第n大的元素排在第n个位置，在它之前的元素均小于它，在它之后的元素均大于它。
不过根据我的实际测试排序方法对于测试时间的影响不是很大。
后面的划分方法反而影响相对更大，我写的划分方案应该相对容易理解，参考题解的划分方法可以自行理解，题解中指出事实上这种最理想的选取最大m*k个元素的方案是恒成立的。

C - Trailing Loves (or L’oeufs?)

题意大概是求出n!在b进制下的后缀0数量。
这题我没有什么好的想法，n!,b的数据范围太大了。
只能参考英文题解。
The problem can be reduced to the following: finding the maximum r that n ! is divisible by b^r.
这句话相当于将题目转化为了质因子分解问题。

以下是我的理解，十进制下末尾0数量来源与这个数能被10^r整除的最大r值，以此类比到b进制，具体的公式推导可以参考英文题解

这里只对使用的对n!求某个质因子的指数的方法做介绍
相比较于原始的累乘其复杂度显然不可行。
这里给出一种比较高效的算法:

n!=1*2*3*……*(n-2)*(n-1)*n

可以表示成所有和m倍数有关的乘积再乘以其他和m没有关系的

=(m*2m*3m*......*km)*ohter     
other是不含m因子的数的乘积   因为 km<=n 而k肯定是最大值  所以k=n/m

=m^k*(1*2*......*k)*other  

=m^k*k!*other*　

从这个表达式中可以提取出m^k，然后按照相同的方法循环下去可以求出k!中因子m的指数。每次求出m的指数的和就是n!中因子m的总指数。
